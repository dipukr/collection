Autoboxing and Unboxing
Automatic conversion of primitive types to corresponding wrapper class is known as autoboxing. Automatic conversion of wrapper class to corresponding primitive types is known as unboxing. Autoboxing and unboxing is performed at compile time by compiler.

Java compiler applies autoboxing when a primitive value:
  • Assigned to a variable of the corresponding wrapper class.
  • Passed as param to a method that expects an object of the corresponding wrapper class.

Java compiler applies unboxing when a wrapper class object:
  • Assigned to a variable of the corresponding primitive type.
  • Passed as param to a method that expects an argument of corresponding primitive type.
Serialization
Serialization is the process of converting an object into stream of bytes. Once we have a stream of bytes we can store that into database, memory or into a file. Serialization is done to persist data or state of an object beyond the life cycle of an application. Second reason to serialize an object is to transmit data or state of an object over a network.

All subtypes of a serializable class are also serializable. Serializable interface has no methods or fields and serves only to identify the semantic of being serializable. Serializability of a class is enabled by the class implementing the io.Serializable interface. Classes that do not implement this interface will not have any of their state serialized or deserialized. 


          User user = new User("Aryan", "aryan@gmail.com", 10)
          FileOutputStream fos = new FileOutputStream("usr.d")
          ObjectOutputStream oos = new ObjectOutputStream(fos)
          oos.writeObject(user)
          oos.close()
          fos.close()
Deserialization
Deserialization is the process of reconstructing an object that has been serialized before. Deserialization is the reverse process of serialization where byte stream
is used to create actual java object into memory. 


          FileInputStream fis = new FileInputStream("usr.d")
          ObjectInputStream ois = new ObjectInputStream(fis)
          User user = (User) ois.readObject()
          ois.close()
          fis.close()

Transient keyword is used to avoid serialization of particular field of an serializable object. Fields marked with transient will not be serialized during serialization. Deserialization process would use the default values for such fields when creating an object out of the serialized form bacause transient fields are not present in the serialized form of an object.
Thread Priority
Every thread has a priority. The thread scheduler uses priority to determine which thread should be allowed to execute. Threads with higher priority are executed in preference to threads with lower priority.

When multiple threads are ready to execute then JVM will selects and executes the Runnable thread which has the highest priority. In case two threads have the same priority then JVM will execute them in FIFO order. 

				
				Thread thread = new Thread("childThread")
				thread.setPriority(Thread.NORM_PRIORITY)
				thread.setPriority(Thread.MIN_PRIORITY)
				thread.setPriority(Thread.MAX_PRIORITY)
Adapter
Adapter pattern is a structural design pattern that allows objects with incompatible interfaces to work together. It acts as a bridge between two incompatible interfaces. Adapter pattern converts the interface of a class into another interface that the client expects. It lets classes work together that couldn’t otherwise because of incompatible interfaces. It catches calls for one object and transforms them to format and interface recognizable by the second object. The adapter pattern convert the interface of a class into another interface clients expect. Sometimes we want two software components with imcompatible interfaces to work together. Adapter pattern lets classes work together that couldn’t otherwise because of incompatible interfaces. 
		interface MediaPlayer {
		    void play(String audioType, String fileName);
		}

		interface AdvancedMediaPlayer {
		    void playVlc(String fileName);
		    void playMp4(String fileName);
		}

		class VlcPlayer implements AdvancedMediaPlayer {
		    public void playVlc(String fileName) {
		        System.out.println("Playing vlc file: " + fileName);
		    }
		    public void playMp4(String fileName) {} // do nothing
		}

		class Mp4Player implements AdvancedMediaPlayer {
		    public void playVlc(String fileName) {} // do nothing
		    public void playMp4(String fileName) {
		        System.out.println("Playing mp4 file: " + fileName);
		    }
		}
